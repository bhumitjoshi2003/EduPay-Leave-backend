## Required Environment Variables

All sensitive config is injected via env vars — there are no defaults for these:

| Variable | Description |
|---|---|
| `DB_URL` | PostgreSQL JDBC URL (e.g. `jdbc:postgresql://localhost:5432/ias_db`) |
| `DB_USERNAME` | DB username |
| `DB_PASSWORD` | DB password |
| `FRONTEND_URL` | Origin for CORS (e.g. `http://localhost:4200`) |
| `MAIL_PASSWORD` | Brevo SMTP password |
| `JWT_PRIVATE_KEY` | Base64-encoded RSA private key (PKCS8) |
| `JWT_PUBLIC_KEY` | Base64-encoded RSA public key (X509) |
| `RAZORPAY_KEY_ID` | Razorpay key ID |
| `RAZORPAY_KEY_SECRET` | Razorpay key secret |

Use `src/main/resources/application-dev.properties` for local dev overrides (JPA `ddl-auto=update`, SQL logging enabled).

To generate a new RSA key pair for local dev: `src/main/java/com/indraacademy/ias_management/util/RsaKeyPairGenerator.java`

## Architecture Overview

Spring Boot 3.4 / Java 21 REST API backed by PostgreSQL. Standard layered architecture: Controller → Service → Repository (Spring Data JPA).

### Authentication & Authorization

- **Stateless JWT** via HTTP-only cookies (`accessToken` cookie, not `Authorization` header).
- `JwtAuthFilter` extracts the token from the cookie on every request.
- Tokens are RS256-signed (RSA key pair). Access token TTL: 60 minutes.
- Roles: `SUPER_ADMIN`, `ADMIN`, `TEACHER`, `STUDENT` (defined in `config/Role.java`).
- Method-level security is enabled (`@EnableMethodSecurity`), so individual endpoints can use `@PreAuthorize("hasRole('ADMIN')")` etc.
- `SecurityUtil` provides `getUsername()` / `getRole()` from the `SecurityContextHolder` — used throughout services for audit logging.
- Public endpoints (no auth): `/api/auth/login`, `/api/auth/refresh-token`, `/api/auth/request-password-reset`, `/api/auth/reset-password`, `/actuator/health`.

### Domain Model

- **`User`** — credentials + role + password-reset token. `userId` is a string PK used as the JWT subject and FK into `Student`/`Teacher`/`Admin`.
- **`Student`** — has `status` (`StudentStatus` enum) and optional bus enrollment (`takesBus`, `distance`).
- **`Teacher`** / **`Admin`** — separate profile tables.
- **`StudentFees`** — monthly fee records per student per academic session. Generated by scheduler.
- **`Payment`** — Razorpay payment records. Stores full fee breakdown (tuition, bus, annual charges, lab, ECA, examination, late fees, platform fee).
- **`FeeStructure`** / **`BusFees`** — configuration tables for fee amounts per class/distance.
- **`Attendance`** — daily attendance records; late/absent days drive `additionalCharges` in payments.
- **`Leave`** / **`AuditLog`** / **`AuditLogArchive`** — leave requests and full audit trail.
- **`Notification`** / **`UserNotification`** — broadcast and per-user notifications.
- **`Event`** — school events with uploaded images.

### Scheduled Jobs

| Scheduler | Cron | Purpose |
|---|---|---|
| `StudentFeesGenerationService` | `0 0 0 1 3 *` (1 March) | Creates `StudentFees` rows for the next academic year |
| `StudentClassProgressionScheduler` | `0 0 0 26 3 *` (26 March) | Promotes active students to the next class |
| `StudentStatusScheduler` | (see class) | Marks students inactive based on `leavingDate` |
| `AttendanceEmailScheduler` | (see class) | Emails attendance summaries |
| `FeeReminderService` | (see class) | Emails fee payment reminders |
| `AuditRetentionService` | (see class) | Archives old audit logs |

### Payment Flow (Razorpay)

1. Client calls `POST /api/payment/create-order` → `RazorpayService.createOrder()` creates a Razorpay order and returns `orderId` + public `razorpayKey`.
2. Client completes payment on Razorpay's UI, then calls `POST /api/payment/verify` with the payment signature.
3. `RazorpayService.verifyPayment()` verifies the HMAC signature, saves the `Payment` record, marks `StudentFees` as paid, updates attendance charge-paid status, sends a confirmation email, and creates an in-app notification.
4. PDF receipts are generated on demand via Apache PDFBox in `PaymentService.generatePaymentReceiptPdf()`.

### Audit Logging

`AuditService.log(...)` is called manually from service methods. `AuditLogArchive` stores older records moved by `AuditRetentionService`. Query with filters via `AuditSpecification` (JPA Specifications).

### File Storage

Event images are stored on the filesystem under `./uploads/events/images/` (configurable via `image.upload.directory`). The path is served as a static resource. `FileStorageProperties` and `FileStorageService` handle this.

### Email

Sent via Brevo SMTP relay. `EmailService` is the single entry point. `app.mail.from` is the sender address (`noreply@edunexify.co.in`).
